# v17.21 Compilation Error Fix Report

**Date:** 2026-05-17 03:16 UTC  
**Error Code:** CT1827 — "Undeclared identifier 'dim'"  
**Issue Count:** 11 errors found and fixed

---

## Problem Analysis

### Root Cause
The Pine Script v17.21 had a **variable declaration order violation**. Multiple color variables were **used before being declared**:

- `_dim` (Inactive/uniform text color)
- `_act` (Active/alert color — lime green)
- `_wrn` (Warning color — orange)
- `_inv` (Invalidated color — red)
- `_sep` (Separator color)
- `_frm` (Frame border color)
- `_lbl` (Label color)
- `_hdr` (Header color)
- `_tr` (Table row/background color)
- `_hudActive` (HUD state boolean)

### Where Errors Occurred

**Usage locations (before fix):** Lines 2508-2555
- Line 2508: `sniperProgressColor = _dim`
- Line 2512: `guardDisplayColor = ... ? _wrn : _dim`
- Line 2516: `confluenceColor = _dim`
- Line 2520: `kzDisplayColor = _dim`
- Line 2524: `poiColor = _dim`
- Line 2528: `momentumColor = _dim`
- Line 2531: `amdColor = _dim`
- Line 2534: `tradeCtxColor = _dim`
- Line 2542: `lastAlertColor = ... ? _act : _dim`
- Line 2579: `signalStageColor = ... ? _act : _dim`
- Line 2583: `sigTodayColor = ... ? _act : _dim`

**Declaration location (before fix):** Lines 2544-2555 (AFTER all uses!)

### Pine Script Requirement
In Pine Script (and most programming languages), **all variables must be declared before they are used**. The v17.21 code violated this principle by trying to use color variables before declaring them.

---

## Solution Applied

### Step 1: Moved Color Variable Declarations Earlier
**From:** Lines 2544-2555 (after all uses)  
**To:** Lines 2501-2512 (before any uses)

Inserted the complete color variable block right after the "Pre-compute display strings" comment:

```pine
// --- v17.20: HUD Colors — MOVED HERE FOR PROPER DECLARATION ORDER ---
bool _hudActive = hudModeInput == "Fixed" or barstate.islast or (last_alert_time > 0 and (timenow - last_alert_time) < 300000)
color _tr  = hudModeInput == "Fixed" ? color.new(#1a237e, 20) : (_hudActive ? color.new(#1a237e, 20) : color.new(#1a237e, 75))
color _hdr = color.new(#0d47a1, 10)      // v17.21: Header row — brighter navy
// v17.21: Text Priority Hierarchy (updated)
color _lbl = color.white                 // Labels: bright white
color _act = color.new(color.lime, 0)    // Active values: bright green
color _dim = color.new(color.silver, 0)  // Inactive: silver
color _wrn = color.new(color.orange, 0)  // Warnings: orange
color _inv = color.new(color.red, 0)     // Invalidated: red
color _sep = color.new(color.gray, 50)   // Separator
color _frm = color.new(color.white, 60)  // v17.21: visible white border
```

### Step 2: Removed Duplicate Declarations
**Deleted:** Lines 2552-2563 (the old/duplicate block that was in the wrong location)
