# SMC Premium/Discount Confluence Engine — v17.22 Completion Report

## Summary
**v17.22 — Gate EQ Line to Active Zone Only** (Visual-Only Update)

### Problem
The EQ (Equilibrium) line was rendering across ALL historical bars via `plot()`, creating a visible squiggle/wave effect as `h4RangeHigh` and `h4RangeLow` values changed over time. The P&D zone boxes only render on the last 50 bars (`bar_index - 50` to `bar_index + 500`), but the EQ line was plotted everywhere `h4RangeHigh`/`h4RangeLow` had values.

### Solution
Gated the `eq_level` value with `bar_index >= last_bar_index - 50` so the EQ line only renders within the active P&D zone region, matching the box's left edge extent exactly.

## Implementation Detail

### Zone Active Gating Variable
The P&D zones don't use a traditional boolean `zoneActive` variable. Instead, the zone boxes are drawn conditionally with:
```pine
if showH4Zones and barstate.islast
    pdPremiumBox := box.new(bar_index - 50, ..., bar_index + 500, ...)
```

The boxes extend from `bar_index - 50` (left) to `bar_index + 500` (right). The EQ line should only be visible in this same region.

### EQ Gating Code (v17.22)
```pine
// EQ Line - 50% midpoint of P&D Zone
eq_level = na(h4RangeHigh) or na(h4RangeLow) ? na : (h4RangeHigh + h4RangeLow) / 2

// v17.22: Only show EQ when within active zone region (last 50 bars matches P&D box extent)
active_eq = (bar_index >= last_bar_index - 50) ? eq_level : na

plot(showH4Zones and showEquilibriumLine ? active_eq : na, title="EQ", color=color.yellow, linewidth=3, style=plot.style_line)
```

### Before vs. After
| Property | v17.21 | v17.22 |
|----------|--------|--------|
| EQ Visibility | ALL historical bars | Last 50 bars only |
| Squiggle Effect | Present (values change per bar) | Eliminated |
| Zone Alignment | Mismatched (EQ everywhere, boxes last 50 bars) | Matched perfectly |
| Plot Budget | 34/64 | 34/64 (unchanged) |
