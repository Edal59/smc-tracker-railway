**Date:** 2026-05-17

## Summary

Replaced the EQ (Equilibrium) line implementation from `plot()` to `line.new()` with a delete-and-redraw pattern. This eliminates the "SMC PD v17:EQ" label from the right price scale and provides a proper flat horizontal line spanning the zone extent.

---

## Zone Start Variable

The P&D zone boxes use `bar_index - 50` as the x1 parameter (left edge). There is no dedicated "zone start" variable — the boxes are drawn with a fixed 50-bar lookback from `bar_index`. The EQ line.new() uses the same `bar_index - 50` to match the zone extent exactly.

---

## Changes Made

### Removed (plot-based EQ)
- `eq_level` variable calculation
- `active_eq` gating variable (50-bar window)
- `plot(showH4Zones and showEquilibriumLine ? active_eq : na, title="EQ", ...)` — **completely deleted**

### Added (line.new-based EQ)
```pine
// v17.23 FINAL EQ FIX — line.new() replaces plot()
var line final_eq_line = na

// 1. Delete the old line every single bar to ensure NO ghosting or squiggles
line.delete(final_eq_line)

// 2. Only draw IF the zone is currently active and valid, and user has enabled it
if showH4Zones and showEquilibriumLine and not na(h4RangeHigh) and not na(h4RangeLow)
    float eq_val = (h4RangeHigh + h4RangeLow) / 2

    // 3. Create a NEW line that spans from the start of the zone to the current bar
    final_eq_line := line.new(
         x1 = bar_index - 50,
         y1 = eq_val,
         x2 = bar_index,
         y2 = eq_val,
         color = color.yellow,
         width = 3,
         style = line.style_solid
         )
```
