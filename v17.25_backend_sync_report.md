## Alert Template Audit (CRITICAL FIX)

### Problem
The old `templates/settings.html` and `tradex-frontend/app/settings/page.tsx` exposed only **2 sample alert blocks** ("Sniper Entry" and "Retrace Entry"), and the `{{plot_X}}` placeholders were **stale and wrong** versus the v17.25 Pine Script. Examples:

| Field | Old template said | v17.25 Pine Script actually emits |
|-------|-------------------|-----------------------------------|
| `h4_bias` | `{{plot_3}}` | `{{plot_10}}` |
| `p_d_zone` | `{{plot_4}}` | `{{plot_11}}` |
| `guardian` | `{{plot_5}}` | `{{plot_13}}` |
| `poi` | `{{plot_6}}` | `{{plot_7}}` |
| `quality` | `{{plot_7}}` | `{{plot_6}}` |
| `confluence` | `{{plot_8}}` | `{{plot_8}}` ✓ |
| `dt_stage` | `{{plot_9}}` | `{{plot_9}}` ✓ |
| `setup_id` | `{{plot_10}}` | (literal `"dynamic"` string, not a plot) |
| Sniper Short entry/SL/TP | (missing) | `{{plot_3}}/{{plot_4}}/{{plot_5}}` |

If a user had copy-pasted the old template into TradingView, **every decoded opportunity row would have been wrong** (wrong bias, wrong zone, wrong guardian).

### Fix Applied
Both `templates/settings.html` and `tradex-frontend/app/settings/page.tsx` now expose **all 4 alert types** with the **exact plot map from v17.25** (verified against `smc_premium_discount_indicator_v17.25.pine` lines 2473-2482):

```
plot_0  → entryLong          plot_7  → poiScore
plot_1  → slLong             plot_8  → confluenceScore
plot_2  → tpLong             plot_9  → dtStage
plot_3  → entryShort         plot_10 → h4Bias
plot_4  → slShort            plot_11 → pdZone
plot_5  → tpShort            plot_12 → kzCode (kill_zone)
plot_6  → qualityScore       plot_13 → guardianCode
```

#### 1. Sniper Long
```json
{
  "api_key": "...",
  "version": "v17.25",
  "type": "sniper_long",
  "setup_id": "dynamic",
  "symbol": "{{ticker}}",
  "entry_price": {{plot_0}}, "stop_loss": {{plot_1}}, "take_profit": {{plot_2}},
  "quality": {{plot_6}}, "poi": {{plot_7}}, "confluence": {{plot_8}}, "dt_stage": {{plot_9}},
  "h4_bias": "{{plot_10}}", "p_d_zone": "{{plot_11}}",
  "kill_zone": "{{plot_12}}", "guardian": "{{plot_13}}",
  "timestamp": "{{timenow}}"
}
```

#### 2. Sniper Short
Same shape as Sniper Long but entry/SL/TP use **plot_3 / plot_4 / plot_5** and `"type": "sniper_short"`.

#### 3. Retrace Long
```json
{
  "api_key": "...",
  "version": "v17.25",
  "type": "retrace_long",
  "setup_id": "dynamic",
  "symbol": "{{ticker}}",
  "suggested_entry": {{plot_0}}, "target_sl": {{plot_1}}, "target_tp": {{plot_2}},
  "poi": {{plot_7}}, "confluence": {{plot_8}},
  "guardian": "{{plot_13}}", "zone": "{{plot_11}}", "kill_zone": "{{plot_12}}",
  "timestamp": "{{timenow}}"
}
```

#### 4. Retrace Short
Same shape as Retrace Long with `suggested_entry/target_sl/target_tp` on **plot_3 / plot_4 / plot_5** and `"type": "retrace_short"`.

An inline "plot index map" cheat-sheet was added above the alert blocks in both the Flask page and the Next.js page so a user can never again paste the wrong placeholder.
