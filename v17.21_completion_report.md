### Phase 3: FIX 2 — EQ Line FINAL ROOT CAUSE FIX

**Root Cause Identified:** Previous versions used `barstate.islast` guard, meaning the EQ line was only drawn on the very last bar. If zone boxes rendered after, or if the chart reloaded, the line could be painted over.

**EXACT Implementation (User's Specified Pattern):**

```pine
// Runs on EVERY bar (no barstate.islast guard)
if showH4Zones and showEquilibriumLine
    float eq_level = (h4RangeHigh + h4RangeLow) / 2.0
    
    if not na(pdEqLine)
        line.delete(pdEqLine)
    pdEqLine := line.new(
         x1 = bar_index - 1,
         y1 = eq_level,
         x2 = bar_index,
         y2 = eq_level,
         color = color.white,
         width = 5,
         style = line.style_solid,
         extend = extend.right
         )
    
    if not na(pdEqLabel)
        label.delete(pdEqLabel)
    pdEqLabel := label.new(
         x = bar_index,
         y = eq_level,
         text = "── EQ",
         color = color.new(color.black, 0),
         textcolor = color.white,
         style = label.style_label_left,
         size = size.normal
         )
```

**Key Changes from v17.20:**
- [x] REMOVED `barstate.islast` guard — now runs on every bar
- [x] x1 = `bar_index - 1`, x2 = `bar_index` (1-bar segment with extend.right)
- [x] `extend = extend.right` for full rightward chart coverage
- [x] `eq_level = (h4RangeHigh + h4RangeLow) / 2.0` — exact 50% midpoint
- [x] Width = 5, color = white, style = solid
- [x] Label uses same delete-and-redraw pattern
- [x] Label text = `"── EQ"` with solid black background
